# ADR 0002: Vancouver Milestone 1 city-artifact backend

**Status:** Accepted at Gate A2 on 2026-07-12

The City of Vancouver public-tree inventory is the specimen baseline. The pinned source is the official whole-dataset export under the Open Government Licence - Vancouver: 185,845 source records retrieved on 2026-07-12, projected to required fields, canonically sorted by `asset_id`, and deterministically gzip-compressed. The raw export is not committed; its SHA-256 is recorded in `data/cities/vancouver/source-manifest.json`.

`npm run city:benchmark` validates its input against the generated artifact **byte hash** and tree/food-site counts, then runs three full-inventory measurements. `city:validate` revalidates the generated and benchmark reports against that exact artifact. It compares the schema-v2 JSON tuple artifact with an actual PMTiles v3 archive containing MVT tiles generated by `geojson-vt`/`vt-pbf` from z0 through z12, decoded through a PMTiles v3 header/directory reader and vector-tile decoder. The compact lookup required for search, inspection, and correct filtering is included in the PMTiles transfer result. PMTiles archive bytes are measured directly: a range-served archive cannot rely on whole-response gzip.

**Decision:** retain one `json-tuple-pack` artifact, `public/cities/vancouver/vancouver.v2.1.0.json`, with embedded species metadata, separately-provenanced site-level `foodSites`, and empty `trails`. The approved city-data compressed-transfer budget is **5,600,000 bytes gzip (5.6 MB)** for the complete inventory; do not change it without another measured approval. JSON measured 5,555,961 bytes gzip. The representative PMTiles z0-z12 archive plus required lookup measured 17,004,527 bytes range-safe transfer and decoded 2,475,303 tile features (185,307 unique IDs); it introduces two geometry/index paths and still must use the worker's exact eligible IDs for displayed filtered cluster counts.

Filter-correct semantics are mandatory for either candidate. JSON sends exactly eligible features to one clustered GeoJSON source. PMTiles must suppress tile-native cluster counts after a committed filter and use the same worker-generated eligible source/counts; hidden records may never contribute to a displayed count. PMTiles is therefore not selected.
