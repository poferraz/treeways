# v2.1.0 Milestone 1 gates

The core-entry baseline is every app-authored module fetched during default startup, including startup workers even if moved to another chunk. The baseline and absolute caps are in `bundle-baseline.v2.1.0.json`; `npm run check:bundle` enforces both the 5% relative and absolute caps.

Performance authority for later rendering work is Pixel 6 on current Chrome at 390x844, with an iPhone 12 Safari smoke check. Use the same fixed camera and filters, warm up for five seconds, collect three 30-second runs, and retain the median frame/long-task/decode/filter-response artifacts under `docs/release-gates/evidence/`. The release gate is median >=55fps with no unreviewed long-task regression. This Milestone does not claim that device test has occurred.

## Gate A2 evidence (approved 2026-07-12)

The complete official Public trees export was pinned on 2026-07-12. Coverage is 185,845 source records, 185,307 accepted, and 538 rejected (531 invalid diameters, 6 invalid heights, 1 invalid coordinate); all 185,307 accepted records retain `canopySpreadM: null`. The normalized schema-v2 artifact contains 1,357 species, 169 access-unknown food sites, and zero trails.

Three-run median evidence is in `data/cities/vancouver/reports/backend-benchmark.json`, self-validated by `city:validate` against the generated artifact byte hash and tree/food-site counts. JSON tuple transfer is 5,555,961 bytes gzip with median decode/index/filter measurements of 56.74/33.16/12.83 ms. The representative z0-z12 PMTiles v3/MVT archive plus the required lookup is 17,004,527 bytes as range-safe archive bytes plus gzip lookup, with 684.95/17.13/10.50 ms. It contains 2,475,303 decoded tile features for 185,307 unique tree IDs. This is a Node benchmark, not the named-device rendering gate.

**Gate A2 decision:** `json-tuple-pack`, a **5,600,000-byte (5.6 MB) compressed city-data cap**, and the filter-correct semantics in ADR 0002 are approved. This decision does not itself authorize Milestone 2.
