# v2.1.0 Milestone 2 evidence certificate

**Status:** complete with no botanical enrichment. Review date: 2026-07-12.

This gate establishes the only permitted path for species/cultivar bloom, forage, harvest, and canopy claims. It does not authorize runtime, rendering, trail, field-guide, deployment, or launch work. The checked-in Vancouver evidence registry is deliberately empty because no reviewed candidate passed every required gate. Unknown/null is the accepted result, not a missing-data fallback.

## Premises

- **[P1] Repository state.** `master` is two commits ahead of `origin/master`; the working tree already contains the Milestone 1 city-pack change set and untracked artifacts. Those changes were preserved. No `AGENTS.md` exists in this repository.
- **[P2] Existing contracts.** `src/domain/forage.js` requires evidence for claimed edibility, bloom, and harvest; bloom requires a 12-bit mask and semantic colour. `src/domain/tree.js` requires `measured`/`estimated` canopy provenance and an estimate method. `src/data/city-schema.js` rejects orphan canopy metadata. Species facts do not contain access or permission fields.
- **[P3] Existing pipeline.** `scripts/city/normalize.js` verifies the pinned snapshot, calls the city adapter, writes normalized data, then `build-pack.js` composes the public artifact. `validate.js` revalidates the exact artifact and report hashes. The Vancouver adapter is the single enrichment join.
- **[P4] Existing evidence.** Gate A2’s accepted snapshot/report counts, deterministic build, PMTiles comparison, and rejection accounting remain the Milestone 1 baseline. The exact artifact hash/report relationship is enforced by `city:validate` and repeated byte identity by `city:verify`.
- **[P5] Candidate sources.** The matrix below records every authority considered in this review.
- **[P6] Assumptions.** None become product data. In particular, municipal taxonomy, common names, food-site labels, genus membership, expected crown spread, and familiar botanical knowledge are not evidence.

## Evidence and implementation traces

- **[T1] Repository trace.** `normalizeCity()` reads `data/cities/<city>/evidence-registry.json`, validates it, and passes it to `vancouverAdapter.normalize()`. The adapter resolves only exact `genus + species + cultivar` records. `buildCityPack()` embeds the registry; `validateCity()` hashes it as part of normalized/artifact drift protection.
- **[T2] Taxonomy trace.** `resolveSpeciesEvidence()` normalizes only whitespace/case in the scientific-name tuple and cultivar. It never reads common name, never performs fuzzy matching, and requires cultivar equality, including both sides being null.
- **[T3] Claim trace.** `validateSpeciesMetadata()` resolves each stable evidence ID and checks that the referenced record contains the exact edibility status/parts, bloom mask/colour, or harvest mask. A nearby claim cannot validate a stronger product field.
- **[T4] Licence trace.** `validateEvidenceRegistry()` requires canonical URL, licence URL/identifier, attribution, and explicit true values for reuse, redistribution, and derivative representation before any source record can be accepted.
- **[T5] Safety trace.** Resolver output has only `edibility`, `bloom`, and `harvest`. It does not return access, permission, preparation, or canopy fields. Unknown edibility cannot have edible parts; conflicts become unknown.
- **[T6] Contradiction trace.** Multiple exact records with different semantics return the unknown/null default. Cultivar variation is not inherited from a species record or another cultivar.
- **[T7] Test trace.** `tests/domain.test.js` exercises exact species/cultivar match, cultivar mismatch, common-name/fuzzy/genus non-match, unsupported parts, masks/colours, unknown harvest, missing provenance/licence, conflicts, canopy method, orphan measurements, giant classification, and access separation. `tests/city-pipeline.test.js` covers deterministic snapshots, rejection accounting, tuple validation, filtered clustering, and artifact packaging.
- **[T8] Artifact trace.** `npm run city:build`, `city:benchmark`, `city:validate`, and `city:verify` operate on the pinned snapshot and exact generated artifact. The A2 cap remains 5,600,000 gzip bytes; this gate must not change the JSON backend, PMTiles decision, or filtering semantics.

## Gate decision

Milestone 2’s evidence-system implementation is accepted only in its conservative state: exact schema and resolver are in place, Vancouver contains no accepted evidence records, and no specimen/site/permission/canopy claim has been inferred. Introducing a candidate source, a licensing policy, a taxonomy authority, a safety interpretation, or any factual enrichment requires a new reviewed matrix row and explicit user direction where it materially changes those choices.
